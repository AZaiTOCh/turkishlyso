# Resgents

**Class:** native agentic form (tokenish / TOKISH taxonomy)  
**Incepted:** v0.5.0 (TOKISH induction)

## Definition

**Resgents** are native agents/agentic units **spawned from** or **programmed into** the local mother codebase — not external agents inserted or plugged into a foreign codebase.

Because they grow from the mother tree (rather than being bolted on), a resgent codebase/count stays **lighter, tighter, leaner, and more specialized**.

**Analogy:** LPUs vs GPUs — specialized lean units vs general external/plugged-in agents.

## Two types

| Type | Name | How it exists |
|------|------|----------------|
| **Reserve Agents** | Spawned from the mother codebase on demand | Ephemeral / callable specialists |
| **Resident Agents** | Programmed into the mother codebase | Always-present specialists (e.g. Gretta, Argus) |

## vs external agents

| | Resgents | External / plugged-in agents |
|--|----------|------------------------------|
| Origin | Mother codebase | Third-party insert/plugin |
| Weight | Lean, specialized | Heavier general runtime |
| Coupling | Native | Boundary / IPC / marketplace |
